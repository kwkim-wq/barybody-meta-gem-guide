🎯 30초 요약
메타가 광고 추천 알고리즘의 "중앙 두뇌"인 GEM이라는 거대 AI를 만들었습니다(ChatGPT를 만든 방식과 같은 규모). 출시 직후 인스타 광고 전환 +5%, 페이스북 피드 광고 전환 +3%를 기록했고, 광고주가 같은 예산으로 더 많은 전환을 얻기 시작했습니다.

대신 마케터의 일은 완전히 바뀌었습니다. 예전처럼 타게팅을 정교하게 짜는 시대는 끝났고, 이제는 AI에게 좋은 재료(크리에이티브)와 깨끗한 신호(전환 데이터)를 공급하는 사람이 이깁니다.

2 GEM이 도대체 뭐예요?

GEM은 메타가 만든 광고 추천용 AI 모델의 이름입니다. ChatGPT 같은 거대 AI를 만드는 방식과 똑같은 규모와 기술로 학습시킨, 현재 광고 업계에서 가장 큰 AI라고 보시면 됩니다.

💡 쉬운 비유
GEM은 직접 광고를 보여주지 않습니다. 대신 "수석 코치" 같은 역할이에요.
메타에는 "광고를 보여줄지 말지" 결정하는 수백 개의 작은 AI가 깔려 있는데(영상 광고용, 메신저용, 피드용 등), GEM은 이 작은 AI들에게 자기 노하우를 가르쳐서 전체 팀을 한꺼번에 더 똑똑하게 만듭니다.

중요한 건 GEM이 사람의 모든 행동을 종합적으로 본다는 점입니다:

  • 📱 페이스북에서 본 영상
  • 📸 인스타에서 누른 좋아요
  • 💬 메신저에서 나눈 대화 패턴
  • 🛒 광고 클릭부터 구매까지의 여정
  • 🖼️ 광고 크리에이티브에 들어있는 이미지 · 글자 · 음성 · 영상

이 모든 정보를 한꺼번에 학습해서, "이 사람이 지금 이 광고에 반응할 가능성이 얼마나 되는가"를 훨씬 더 정확하게 예측합니다.

멀티모달 학습이란?

GEM의 핵심 특징 중 하나는 멀티모달(Multimodal) 학습입니다. "멀티(Multi) = 여러 가지", "모달(Modal) = 형태"라는 뜻으로, 쉽게 말하면 텍스트·이미지·영상·행동 데이터를 따로따로가 아니라 한꺼번에 연결해서 본다는 의미입니다.

📺 비유: 광고를 "보는" AI vs 광고를 "읽는" AI

예전 AI는 마치 숫자만 보는 회계사 같았습니다.
"이 광고 → 클릭 327번, 구매 12번" — 숫자만 기록하고, 광고 안에 뭐가 있는지는 신경 쓰지 않았습니다.

GEM은 광고를 직접 보고 읽고 듣습니다.
영상 속 배경이 실내인지 야외인지, 카피가 감성적인지 기능 설명인지, 출연자가 있는지 제품만 나오는지 — 이런 요소들을 사람의 클릭·구매 행동과 연결해서 패턴을 찾아냅니다.

📅 언제부터 적용됐나 — 메타 광고 AI 진화 타임라인

많은 마케터들이 "2024년 말부터 광고가 이상해졌다"고 느끼는데, 사실 메타는 3단계에 걸쳐 광고 시스템을 통째로 교체해왔습니다. 어느 시점에 무엇이 바뀌었는지 알면, 지금 우리가 겪는 변화가 어디까지 진행됐는지 가늠할 수 있습니다.

4 메타 광고 시스템 전체 그림

많은 사람이 "GEM = 새 알고리즘"이라고 생각하지만, 실제로는 3개의 AI가 순서대로 작동하는 3단계 시스템입니다. 내 광고가 사용자에게 도달하기까지 이 3단계를 모두 통과합니다.

단계이름하는 일마케터 관점
1단계 Andromeda
안드로메다 · 후보 선별		 수백만 개 광고 중 사용자 한 명당 약 1,000개 후보를 골라냄. 이때 광고의 이미지·영상·카피·음성을 직접 읽어서 매칭. 이 단계를 통과해야 경쟁에 올라갈 수 있음. 크리에이티브가 Andromeda에 "읽히지 않으면" 노출 자체가 안 됨.
2단계 GEM
젬 · 정밀 평가		 1,000개 후보를 정밀하게 점수화. 사용자가 구매 여정의 어느 단계에 있는지, 전환 가능성이 얼마나 되는지 예측. GEM 학습의 질이 곧 광고 효율. 신호 데이터(CAPI·전환 이벤트)와 크리에이티브 다양성이 GEM의 연료.
3단계 Lattice
래티스 · 노출 결정		 피드·스토리·릴스 등 Meta의 모든 면(surface)을 하나의 모델로 통합 랭킹. 어디에 어떤 형식으로 보여줄지 결정. 플레이스먼트 자동화(자동 노출 위치)가 권장되는 이유. Lattice가 각 자리별로 알아서 최적화함.

Andromeda에 "읽히려면" 어떻게 해야 하나?

1단계 Andromeda를 통과하지 못하면 GEM·Lattice 단계 자체에 올라가지 못합니다. 수집된 자료에서 확인된 조건은 두 가지입니다.

조건내용출처
유사 소재는 Entity ID로 묶임 Andromeda는 픽셀 단위 분석 + 자연어 처리(NLP)로 광고 간 유사성을 감지합니다. 비슷한 소재는 Entity ID로 묶여서 개별 노출 기회를 못 받습니다. 색깔·카피 한 줄 정도의 변형은 "다른 소재"로 인식되지 않습니다. Flâneur Archives
카피·비주얼을 의미론적으로 읽는다 Andromeda는 광고의 이미지·영상·카피를 읽고 "이 광고가 어떤 맥락인지" 파악합니다. 단순 키워드 매칭이 아니라 의미(Semantic) 단위로 이해합니다. 소재 안의 언어·상황이 구체적일수록 Andromeda가 더 명확하게 읽습니다. Flâneur Archives,
Silverback Strategies
소재 자체가 타겟팅 신호 Andromeda는 광고 안의 언어·상황·인물을 읽고 적합한 사람을 찾아냅니다. 타게팅 옵션에서 "30대 여성"을 설정하는 게 아니라, 소재 안에 그 사람들의 맥락을 담으면 Andromeda가 알아서 찾아갑니다. 마케터가 미처 생각 못 한 유사 그룹까지 확장해서 찾아주기도 합니다. Flâneur Archives,

이제부터 살펴볼 4가지 영역(콘텐츠 · 세팅 · 운영 · 타게팅)이 모두 이 변화를 반영해서 바뀌어야 합니다.

🤔 왜 그래야 하나? — 핵심 질문 6가지

앞에서 "이렇게 바뀌었다"고 했지만, "왜 그런지"를 모르면 결국 응용을 못 합니다. 마케터들이 가장 많이 묻는 6가지 "왜?"를 정리했습니다.

❓ Q1. "깨끗한 전환 데이터"가 왜 그렇게 중요한가요?

✅ 맞습니다. 정확히 짚으셨어요. AI가 "광고 클릭 → 장바구니 → 결제 시작 → 구매"라는 한 줄의 완전한 여정을 정확히 확인할 수 있어야 학습이 가능합니다.

🧠 AI 입장에서 데이터는 "교과서"

GEM이 학습하는 방식을 단순화하면 이렇습니다:

"30대 여성 A씨가 이 광고를 보고 → 3초 시청 → 클릭 → 장바구니 담음 → 결제 시작 → 결국 구매"

이런 완전한 데이터 한 줄이 100만 개, 1,000만 개 쌓이면 GEM이 "이런 광고는 이런 사람에게 통한다"는 패턴을 학습합니다.

⚠️ 신호가 더러우면 무슨 일이 생기나

  • 커머스 웹사이트의 픽셀 데이터는 평균 30~50%가 누락됨
  • AI는 "절반의 정답"만 보고 학습 → 잘못된 사람에게 광고 노출
  • 예: 실제로는 구매했는데 메타가 "구매 안 함"으로 인식 → AI가 "이 광고 별로네"라고 잘못 학습
  • 결과: 진짜 좋은 광고가 줄어들고, 효과 없는 광고에 예산이 흘러감

❓ Q2. "크리에이티브 + 깨끗한 신호" 2가지가 왜 승부처인가요?

통제할 수 있는 영역이 메타와 마케터 사이에서 명확히 갈라졌기 때문입니다.

통제 주체통제 영역
🤖 메타(AI) 누구에게 / 언제 / 어디에 / 얼마에 보여줄지 (= 모든 타게팅·입찰·플레이스먼트)
🧑‍💼 마케터 ① 무엇을 보여줄지 (크리에이티브)
② 어떤 결과 데이터를 줄지 (신호)

🎨 AI 입장에서 크리에이티브 = "타게팅 도구"

Andromeda는 이미지·영상·카피·음성 자체를 읽고 "이 광고가 어떤 사람에게 어울리는지" 매칭합니다. 그래서 크리에이티브 = AI가 사용하는 타게팅 무기입니다. 무기가 적으면 AI가 정확하게 못 쏩니다.

📊 AI 입장에서 신호 = "교과서/정답지"

어떤 광고가 진짜로 효과 있었는지, 어떤 사람이 진짜로 구매했는지 알려주는 데이터입니다. 교과서가 더러우면 학습이 망가집니다.

🎯 핵심 — 이 둘 중 하나라도 부족하면 AI가 학습을 못 합니다.
• 크리에이티브 부족 → AI에게 타게팅할 도구가 없음 → 정확도 ↓
• 신호 부족·더러움 → AI가 학습할 정답이 없음 → 정확도 ↓

예전에는 "타게팅 세팅"이 마케터의 승부처였지만, 이제 그건 AI가 가져갔습니다. 남은 2가지가 곧 마케터의 전부입니다.

❓ Q3. 마케터는 이 2가지를 어떻게 대처해야 하나요?

🎨 A. 크리에이티브 측면 — "양과 다양성"

  • ✅ 크리에이티브 생산을 시스템화 — 매주 2~3개 신규 소재 투입 루틴 만들기
  • ✅ AI 도구 적극 활용 — Midjourney, Runway, Sora, 메타 생성 도구 등으로 생산 속도 5~10배 향상
  • ✅ UGC 크리에이터 풀 구축 — 외부 크리에이터·고객 후기 영상 정기 확보
  • ✅ "1개의 완벽한 영상"보다 "10개의 다양한 시도"
  • ✅ 4가지 유형 골고루 — 제품·사회적 증거·오퍼·브랜드 스토리
  • ✅ 9:16 세로 영상 비중 40% 이상 — 릴스·스토리가 메인 인벤토리
  • ✅ 빈도(frequency) 지표 기반 로테이션 — 빈도 2.5+ 주의, 3.0+ 교체 (업계 평균은 약 10일 전후)
  • ✅ 내부 콘텐츠 디렉팅 역량 내재화 — 외주만으로는 속도가 안 따라옴

📊 B. 데이터 신호 측면 — "양과 품질"

  • ✅ CAPI(Conversions API) 도입 — Safari ITP·인앱 브라우저·광고 차단기 등으로 누락되는 30~50% 데이터 보완. 모르면 개발자·대행사에 의뢰.
  • ✅ EMQ(Event Match Quality) 9.0 이상 유지 — 광고 매니저 → 데이터 소스에서 확인 가능
  • ✅ 고객 정보 최대한 수집·전송 — 이메일·전화번호·이름·생년월일·우편번호 (해시 처리되어 전송됨)
  • ✅ 마이크로 전환 함께 전송 — 페이지뷰·장바구니 추가·결제 시작도 함께 보내 신호 밀도 보강
  • ✅ 도메인 인증·픽셀 코드 깨끗하게 유지 — 중복 발화·오류 점검 정기적으로
  • ✅ 광고세트당 주 50건 이상 전환 확보 예산 — 신호 양 자체가 부족하면 품질도 무의미
  • ✅ 오프라인 전환도 업로드 — 매장 구매, 전화 주문 등도 메타에 업로드 가능
💡 현실적 우선순위
① 이번 달 안에: CAPI 도입 + 마이크로 전환 전송 세팅
② 다음 달까지: 크리에이티브 생산 루틴(월 8~12개 신규) 안착
③ 분기 단위로: AI 생성 도구 + UGC 크리에이터 풀 확장

❓ Q4. 좁은 관심사·세밀한 유사대상 타게팅이 왜 성과가 떨어지나요?

🚫 4가지 이유

  1. AI에게 손 묶기
    Andromeda는 수만 가지 신호(행동·맥락·크리에이티브 매칭)를 보고 적합한 사람을 찾습니다. 좁은 타게팅 = "이 풀에서만 뽑으라"고 손을 묶는 것 → 더 좋은 고객을 놓침.
  2. 신호 밀도 부족 → 학습 제한
    좁은 풀에서는 주 50건 전환을 채우기 어려움 → GEM이 "학습 제한" 상태에 머무름 → 학습 자체가 시작 안 됨.
  3. 좁은 풀에서 경쟁 심화 → CPM 상승
    같은 좁은 오디언스를 다른 광고주도 노림 → 입찰가 상승 → 같은 예산으로 노출 ↓.
  4. 이중 좁히기의 비효율
    Andromeda가 이미 크리에이티브로 정확히 매칭하는데, 거기에 관심사로 또 좁히면 이중 필터 → 효율 급락.
🎯 비유 — AI에게 "이 동네에서만 고객을 찾아"라고 시키는 것과 같음. 정작 옆 동네에 더 좋은 고객이 있는데 발견 못 함.

❓ Q5. 마이크로 A/B 테스트(버튼 색·카피 한 줄)가 왜 의미 없나요?

🔍 Entity ID 그룹화 때문

Andromeda는 비슷한 크리에이티브를 "Entity ID"라는 같은 ID로 묶어서 처리합니다.

  • 같은 영상에 CTA 색만 다른 광고 → 시스템 입장에서 "같은 광고"
  • 같은 이미지에 헤드라인 한 줄만 다른 광고 → "같은 광고"
  • 결과: 두 광고가 경쟁하는 게 아니라 하나의 묶음으로 노출됨
  • 의미 있는 비교 데이터를 얻을 수 없음 → A/B 테스트의 본질이 망가짐

✅ 진짜 A/B 테스트는 이렇게

본질적으로 다른 것끼리 비교해야 의미 있는 학습이 일어납니다:

  • 훅(첫 3초)이 본질적으로 다름
  • 출연진이 다름 (모델 vs UGC 크리에이터 vs 창업자)
  • 메시지 각도가 다름 (기능 강조 vs 감성 어필 vs 사회적 증거)
  • 시각 스타일이 다름 (스튜디오 촬영 vs 일상 핸드폰 촬영)
  • 포맷이 다름 (영상 vs 캐러셀 vs UGC 후기)
💡 실무 팁 — 광고 1개를 만들 때 "이 광고와 본질적으로 어떻게 다른가?"를 자문해 보세요. CTA 색·카피 한 줄 차이라면 그냥 같은 광고입니다.

❓ Q6. 크리에이티브 부족이 왜 "가장 큰 성과 발목"인가요?

🧠 GEM은 재료가 많을수록 잘 학습하는 AI

  • 크리에이티브 2~3개 → AI가 패턴 발견 불가 (샘플 수 부족)
  • 광고세트당 10~15개 이상부터 → AI가 의미 있는 학습 시작
  • 20~50개 → 상위 광고주들이 이 구간에서 ROAS 최대화

📈 다양성 = "노출 기회의 수"

Andromeda는 Entity ID 단위로 노출 기회를 배정합니다. 즉:

  • 비슷한 광고 30개 → Entity ID 1개 → 노출 기회 1개
  • 본질적으로 다른 광고 10개 → Entity ID 10개 → 노출 기회 10개

즉 "크리에이티브를 다양화한다"는 건 단순히 변화를 주는 게 아니라 "노출 받을 자격의 수를 늘리는 일"입니다.

🎯 ROAS 상한이 낮아지는 이유

크리에이티브가 부족하면 — AI가 알아낼 수 있는 인사이트의 상한이 낮아짐 → ROAS의 천장도 낮아짐.

반대로 크리에이티브가 풍부하면 — AI가 "이 페르소나에는 이 광고가, 저 페르소나에는 저 광고가 통하더라"는 정교한 매칭을 학습 → ROAS 상한이 올라감.

그래서 메타 본사가 "Advantage+ 크리에이티브 켜면 ROAS 22% 상승"이라고 발표한 것입니다.
🚀 결론 — GEM 시대의 ROAS는 "내가 매주 만드는 신규 크리에이티브 수"에 거의 비례합니다. 타게팅·입찰·세팅을 아무리 만져봐야 천장이 정해져 있고, 그 천장을 올리는 유일한 레버가 크리에이티브 양산입니다.

🎨 ① 콘텐츠(소재) — 가장 중요해진 영역

GEM 시대에 가장 큰 성과 레버는 크리에이티브입니다. 메타가 발표한 자료에서도, 모든 업계 전문가의 결론도 같습니다. 타게팅이 AI에게 넘어간 만큼, 크리에이티브가 유일한 통제 가능 변수가 됐습니다.

01핵심 원칙 1: 소재 수량 — "5~15개 최적" 법칙

💡 최적 구간: 광고세트당 5~15개. 이 범위가 AI의 "탐색(다양성)"과 "집중(학습)"을 동시에 가능하게 합니다.
  • 광고세트당 5~15개의 서로 다른 크리에이티브 — AI가 충분한 다양성을 탐색하면서도 각 소재에 의미 있는 데이터를 쌓을 수 있는 범위
  • 단, 여러 세트로 쪼개지 말고 1개 세트에 통합: 연구 결과 5개 세트×5개 소재보다 1개 세트에 소재 25개를 통합했을 때 전환율 17%↑, CPA 16%↓
⚠️ "노이즈 역설(Noise Paradox)" — 소재를 30개 이상 넣으면 오히려 역효과
한정된 일일 예산(예: 20~80만원)이 30개로 분산되면, 소재당 노출이 너무 적어 승자를 가릴 수 없습니다. 잠재력 있는 소재가 데이터도 못 받고 사장됩니다.
예외: 일일 광고비 $10,000(약 1,400만원) 이상 대형 계정은 20개+ 로테이션 가능

02핵심 원칙 2: "진짜 다른" 크리에이티브 — 의미론적 다양성

메타의 Andromeda는 비슷한 크리에이티브를 한 묶음으로 인식합니다(Entity ID라고 부름). 단순히 색깔·배경을 바꾸는 시각적 변형은 중복(Redundancy)으로 처리됩니다. Andromeda가 원하는 건 의미론적 다양성(Semantic Diversity)입니다.

✅ 이건 다른 크리에이티브 (의미 차이 있음)
  • 훅(첫 3초)이 완전히 다름
  • 출연진이 다름 (모델 vs UGC)
  • 메시지 각도가 다름 (기능 강조 vs 감성 어필)
  • 시각 스타일이 다름 (스튜디오 vs 일상)
❌ 이건 같은 크리에이티브로 묶임
  • CTA 버튼 색깔만 변경
  • 제목 카피 한 줄 수정
  • 같은 영상에 자막만 추가
  • 이미지 위치만 살짝 변경
🚫 복제 전략의 종말 — "성공한 광고"를 살짝 변형해서 복제하는 방식은 이제 역효과입니다.
Andromeda는 사용자가 유사 광고에 반복 노출될 때 흥미가 떨어지는 속도를 수학적으로 모델링(관심 붕괴 곡선, Interest Decay Curve)합니다. 비슷한 구조의 광고가 계속 노출되어도 전환이 없으면, 해당 광고뿐 아니라 유사한 구조의 다른 광고들 랭킹까지 일괄 하향 조정합니다.

03핵심 원칙 2-1: P.D.A 프레임워크 — 소재 기획의 기준

진정한 다양성을 확보하기 위한 실전 프레임워크입니다. 서로 다른 P.D.A 조합으로 소재를 기획하면, 시스템 입장에서 완전히 다른 타겟 그룹을 공략하는 무기가 됩니다. 이렇게 구성된 소재는 89%의 확률로 새로운 오디언스를 잠금 해제한다는 연구 결과가 있습니다.

요소의미예시 대비
P — Persona
(대상/상황)		 광고 속 주인공·상황을 다르게 설정 "야근하는 직장인" vs "육아 중인 부모"
D — Desire
(욕구/소구점)		 강조하는 혜택·감정을 다르게 설정 "시간 절약(효율)" vs "사회적 인정(자랑)"
A — Awareness
(인지 단계)		 고객이 제품을 얼마나 아는지에 맞게 설정 "문제 제기형(모르는 사람)" vs "해결책 비교형(알아보는 사람)"
💡 실전 적용 — "30대 워킹맘"을 타겟으로 하고 싶다면, 타게팅 옵션에서 설정하는 게 아니라 광고 카피·영상 안에 그들의 일상·언어·고민을 담으세요. Andromeda가 크리에이티브를 읽고 자동으로 그 사람들을 찾아줍니다.

06핵심 원칙 5: 첫 3초가 모든 것을 결정

🎬 영상은 첫 3초 안에 시선을 잡지 못하면 끝입니다. 메타가 측정하는 새 지표:
• Hook Rate = 3초 이상 본 사람 / 광고 본 사람 (목표: 30% 이상)
• Hold Rate = 15초 이상 본 사람 / 광고 본 사람 (목표: 10% 이상)
강한 질문, 충격적인 비주얼, 명확한 가치 제안으로 시작하세요.

07핵심 원칙 6: 로테이션 주기 — "빈도 지표" 기반으로 판단

흔히 "10~14일마다 교체"라고 알려져 있지만, 이건 메타 공식 가이드가 아니라 업계 평균 데이터입니다. 메타 공식 방식은 고정 사이클이 아니라 빈도(frequency) 지표 기반 트리거입니다.

빈도(Frequency) 지표상태액션
~ 2.0🟢 양호그대로 유지
2.5 이상🟡 주의새 소재 준비 시작
3.0 이상🔴 위험즉시 교체 권장
빈도 ↑ + CTR ↓ + CPM ↑ 동시 발생🚨 명백한 피로즉시 교체

참고 — 업계 평균 데이터 (Motion App, AdStellar 등 분석):

  • 상위 브랜드 평균: 10.4일마다 로테이션
  • 퍼널 상단(신규 고객 타게팅): 5~7일마다 새 시각자료
  • 퍼널 중간(리타게팅): 10~14일까지 늘려도 OK
  • 저예산 계정: 2~4주 사이
💡 실무 권장 — 고정 사이클에 집착하지 말고 광고 매니저에서 빈도·CTR·CPM 지표를 보면서 교체 시점을 판단하세요. 다만 매주 2~3개 신규 소재 추가 루틴은 유지해야 GEM이 학습할 새 재료가 끊기지 않습니다.

08핵심 원칙 7: 필터링 크리에이티브 — "좋은 신호"를 주는 소재

예전엔 자극적인 밈·낚시성 문구로 CTR을 높이는 방식이 통했습니다. 지금은 역효과입니다. 구매 의도 없는 사람들이 대거 클릭하면, GEM은 "이 광고의 고객은 구경꾼"이라고 학습하고 더 많은 구경꾼에게 광고를 뿌립니다.

✅ 필터링 크리에이티브 (GEM이 좋아함)
  • 제품 가격을 광고에 명시
  • 구체적인 혜택·스펙을 전면에 배치
  • "이런 분께 맞습니다"처럼 타겟을 광고 안에서 명확히 정의
  • 초반에 CTR이 낮더라도 CVR(전환율)이 높은 소재
❌ 낚시성 크리에이티브 (학습을 망침)
  • 자극적 문구로 클릭 유도, 랜딩에서 실망
  • CTR은 높지만 구매로 이어지지 않음
  • 단순 호기심 클릭을 유발하는 밈·논란성 소재
⚠️ CTR 함정 주의 — CTR이 높다고 좋은 소재가 아닙니다.
CTR이 높아도 전환율(CVR)이 낮으면 광고비만 소진하고 GEM을 잘못 학습시킵니다. CTR은 반드시 CPA·ROAS·CVR과 함께 복합적으로 판단하세요. 클릭만 높고 전환이 없는 소재는 빠르게 중단해야 합니다.

⚙️ ② 광고 세팅 — 단순할수록 잘 돈다

예전에는 캠페인을 잘게 쪼개고 세팅을 정교하게 짜는 게 잘 하는 거였습니다. 지금은 정반대입니다. 캠페인이 단순하고 통합되어 있을수록 AI가 학습할 데이터가 많아져서 성과가 좋아집니다.

01캠페인 구조 — "싱글 버킷" 전략

  • ✅ 각 캠페인 안에서 광고세트는 1~2개로 통합
  • ✅ Advantage+ 쇼핑 캠페인 우선 사용 (메타가 가장 많이 밀어주는 캠페인 유형)
  • ❌ 같은 제품을 타겟 연령대별·관심사별로 캠페인을 쪼개기 → 모든 캠페인이 학습 부족 상태에 빠짐
  • ✅ 제품 A vs 제품 B처럼 제품이 달라서 나누는 건 정상 — 아래 참고

02최적화 목표 — 가장 하단으로

  • 가능하면 구매(또는 리드)로 최적화
  • "링크 클릭", "랜딩 페이지 조회" 같은 상위 단계 최적화는 GEM의 학습 능력을 다 활용하지 못함
  • 전환이 너무 적은 신규 계정만 임시로 상위 단계 사용, 전환 쌓이는 즉시 변경

03예산 — "신호 밀도"를 위한 최소 기준

💰 황금 공식 ①: 광고세트당 주당 50건 이상의 전환
이게 안 되면 AI가 학습 제한 상태에 머무릅니다.

주간 최소 예산 계산법: 목표 CPA × 50 = 주당 필요 예산
• 목표 CPA 3만원 → 주당 150만원 (일 21만원 이상)
• 목표 CPA 5만원 → 주당 250만원 (일 36만원 이상)
• 이 미만이면 광고세트를 줄이고 예산을 합치는 게 우선
💰 황금 공식 ②: 일일 예산 = 목표 CPA × 3
Andromeda는 초기 반응이 저조한 소재를 빠르게 도태시킵니다. 소재가 충분한 기회를 얻기 전에 예산 부족으로 꺼지는 것을 막으려면, 하루에 최소 3건 이상의 전환이 발생할 수 있는 예산이 필요합니다.

• 목표 CPA 3만원 → 일일 최소 9만원
• 목표 CPA 5만원 → 일일 최소 15만원
• 예산이 이보다 작으면 먼저 광고세트 수를 줄이고 예산을 합치세요.

04예산 규모별 권장 소재 수

캠페인당 소재를 몇 개 넣어야 할지는 예산에 따라 달라집니다. 캠페인당 일 예산 ÷ (목표 CPA × 3) = 소재 상한선이 기준입니다.

캠페인당 일 예산권장 소재 수이유
100만원 이상 12~15개 소재당 충분한 예산 확보, 다양성 극대화 가능
40~99만원 8~12개 적정 탐색·집중 균형 구간
20~39만원 5~8개 소재 수 더 늘리면 소재당 데이터 부족
20만원 미만 3~5개 캠페인 통합 우선 검토 권장
💡 계산 예시 — 목표 CPA 3만원, 월 3,000만원(일 100만원) 지출, 캠페인 2개 운영

캠페인당 일 예산: 50만원
소재당 최소 필요 예산: 3만원 × 3 = 9만원
소재 상한선: 50만원 ÷ 9만원 ≈ 5개 (이론상 한계)
실제 권장: 8~10개 — 모든 소재가 매일 동일하게 예산을 받는 게 아니라 성과 좋은 소재에 몰리기 때문에, 상한선보다 여유 있게 운영 가능
⚠️ 위 표의 권장 수치는 "목표 CPA × 3" 법칙(출처: Flâneur Archives)과 "주당 50건 전환" 기준(출처: Meta 공식)을 토대로 계산한 가이드입니다. 실제 목표 CPA에 따라 달라지므로 반드시 본인 계정 기준으로 다시 계산하세요.

05제품이 여러 개일 때 — 캠페인을 어떻게 나눌까?

"캠페인 통합"은 같은 제품을 타겟·연령·관심사별로 쪼개지 말라는 뜻입니다. 제품이 달라서 나누는 건 다른 이야기입니다 — 제품마다 목표 CPA·마진·예산 배분이 다르기 때문에 캠페인을 분리하는 게 맞습니다.

💡 판단 기준: 제품별로 주당 50건 전환을 만들 수 있는가?

상황 권장 구조 이유
제품별로 주당 50건+ 전환 가능 ✅ 제품별 캠페인 분리 예산 통제 가능, 각 캠페인이 충분한 신호를 독립적으로 학습
판매량이 적어 주당 50건이 어려운 제품이 섞여 있음 ✅ 소규모 제품들은 하나의 캠페인에 통합 전환 신호를 합쳐야 AI가 학습 가능, 분리하면 모두 학습 부족 상태
전 제품의 예산이 소규모 (월 300만원 이하) ✅ 전체 1개 캠페인 + 소재로 제품 구분 예산을 쪼개면 어떤 캠페인도 학습 못 함

GEM은 학습하는 데 시간이 필요합니다. 예전처럼 매일 캠페인을 켜고 끄고 수정하면 학습이 매번 리셋됩니다. 2026년 메타 광고 운영의 핵심은 "참는 힘"입니다.

01학습 기간 — 7일 또는 50 전환까지 손대지 말기

  • 메타 공식 권장이 3~4일에서 7일 이상으로 변경됨
  • 광고세트가 50 전환을 모으기 전까지는 평가 금지
  • 학습 단계에서 끄거나 큰 변경 → 처음부터 다시 학습 → 예산 낭비

02변경 빈도 — 적을수록 좋다

✅ 해도 되는 변경
  • 새 크리에이티브 추가
  • 예산 20% 이내 증액
  • 성과 나쁜 광고만 끄기
  • 주 단위 ROAS 검토
❌ 학습을 망치는 변경
  • 일일 단위 캠페인 끄기·켜기
  • 예산 50% 이상 급변
  • 최적화 목표 변경
  • 타게팅·플레이스먼트 잦은 수정

  • 📅 3~7일 롤링 평균으로 판단
  • 오늘 ROAS가 낮다고 광고를 끄지 말 것 (다음날 회복하는 경우 많음)
  • 크리에이티브 단위로는 최소 1주일 데이터 확보 후 판단

04측정 지표 — ROAS만 보지 말기

플랫폼 ROAS는 어트리뷰션 윈도우가 동적으로 바뀌어서 신뢰도가 떨어졌습니다. 새로운 측정 체계가 필요합니다.

05신호 데이터 품질 — "음식이 안 좋으면 AI도 굶는다"

  • 🔌 CAPI(Conversions API) 설정 필수 — 모르면 개발자나 대행사에 요청하세요. Safari ITP·인앱 브라우저·광고 차단기·3자 쿠키 차단 등으로 픽셀만으로는 데이터 30~50%가 사라집니다 (커머스 웹사이트도 동일).
  • 📊 Event Match Quality(EMQ) 점수 9.0 이상 목표 — 메타 광고 매니저 → 데이터 소스에서 확인 가능. 9.3 달성 시 CPA 약 18% 감소.
  • 🛒 구매 외에 장바구니 추가, 결제 시작 같은 마이크로 전환도 함께 보내기

🎯 ④ 타게팅 설정 — 가장 큰 패러다임 변화

이 영역의 변화가 가장 충격적입니다. "좁은 타게팅이 더 정확하다"는 옛 상식이 정반대로 뒤집혔습니다. 이제 메타는 광고주가 입력한 타게팅이 아니라, 크리에이티브 자체를 읽고 적합한 사람을 찾아냅니다.

01기본 원칙: 광범위 타게팅

  • ✅ 관심사·행동 타게팅 빼기 — Andromeda가 크리에이티브로 더 정확하게 타게팅합니다.
  • ✅ 유사 대상(Lookalike) 거의 불필요 — GEM이 비슷한 사람을 알아서 찾아냄.
  • ✅ 제외 타게팅 최소화 — 정말 필요한 경우(이미 구매한 고객 제외 정도)만.
  • ✅ 위치·연령·성별만 큰 틀로 설정 — 예: "한국 전체, 25~55세" 정도면 충분.

02왜 좁은 타게팅이 망하나?

🚨 좁은 타게팅의 3가지 문제
① AI에게 "이 풀에서만 찾으라"고 손을 묶는 것 — 더 좋은 고객을 못 찾음
② 신호 밀도가 떨어져 학습이 정체됨 (주당 50 전환을 못 채움)
③ CPM이 올라감 (좁은 풀에서 경쟁이 심해짐)

03타게팅은 "크리에이티브"로 한다

예를 들어 "30대 워킹맘"을 타게팅하고 싶다면, 광고 카피와 영상에 "30대 워킹맘"의 일상·고민·언어를 담으세요. 그러면 Andromeda가 그 카피를 읽고 자동으로 30대 워킹맘에게 보여줍니다. 게다가 마케터가 미처 생각 못 한 "30대 워킹맘처럼 비슷한 구매 패턴을 가진 다른 그룹"까지 찾아내 줍니다.

✅ 새로운 타게팅 방식
  • 광범위 타게팅 + 페르소나별 크리에이티브 4~5세트
  • 각 페르소나의 언어·시각·관심사를 광고 안에 녹임
  • AI가 학습할 시간을 줌
❌ 옛날 방식 (이제 안 통함)
  • 관심사 10~20개 스택해서 좁은 풀 만들기
  • 유사대상 1%, 2%, 3% 잘게 쪼개기
  • 일일 데이터 보고 타게팅 미세 조정

04리타게팅은?

  • 여전히 유효하지만, 별도 캠페인보다 Advantage+ 안에 통합하는 것이 트렌드
  • 웹사이트 방문자, 장바구니 이탈자 풀은 만들어둘 가치 있음
  • 다만 전체 예산의 20~30% 이내로만 — 메인은 신규 고객 확보

05신규 고객 확보 신호 활용

  • 메타에 "신규 고객 우대"(New Customer Optimization) 옵션이 있으면 활성화
  • CRM 고객 리스트를 업로드해서 "기존 고객 제외"로 활용 가능
